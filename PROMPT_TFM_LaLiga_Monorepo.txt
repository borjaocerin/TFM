### 2) Backend (FastAPI + Python 3.11)
**Objetivo**: exponer endpoints para **ingestión**, **features**, **entrenamiento**, **predicción** y **comparativa con cuotas**.

#### 2.1 Dependencias clave
- FastAPI, Uvicorn, Pydantic, Pandas, NumPy, Scikit‑learn, XGBoost (opcional), Joblib  
- Metrics & plots: scikit‑learn calibration, reliability; matplotlib (guardar png en `/static`)  
- Validación de datos: `pandera` (opcional)  
- Programación: `APScheduler` para jobs (re‑entrenos, refresco datasets)

#### 2.2 Modelo ML (multiclase 1X2)
- **Etiquetas**: `H=0`, `D=1`, `A=2`.  
- **Particionado**: `TimeSeriesSplit`.  
- **Métricas**: `log_loss`, `brier`, `ECE`, `accuracy`, `f1_macro`.  
- **Selección**: ordenar por `log_loss` (menor) y desempatar por `accuracy` (mayor).  
- **Calibración**: Platt (`sigmoid`) o **Isotónica** → guardar curvas.  
- **Persistencia**: `model.pkl` + `metadata.json` (fecha, features, métricas, filas entrenadas).

#### 2.3 Features (capas)
- **Nivel partido**: `xg_home/away`, `xga_home/away`, `poss_*`, `sh_*`, `sot_*`, etc.  
- **Diferenciales**: `xg_diff`, `poss_diff`, `sh_diff`, …  
- **Rolling por equipo (estrictamente t−1)**: medias móviles `last5` y `last10` para `xg/xga/poss/sh/sot/gf/ga/points`.  
- **ELO** (opcional): `elo_home`, `elo_away`, `elo_diff` usando snapshot anterior a la fecha.

#### 2.4 Esquemas (Pydantic) y contratos principales
- **POST `/api/v1/datasets/ingest`**  
  Entrada: rutas de CSV locales (`historical`, `football_data_dir`, `elo_csv?`)  
  Acción: normaliza nombres, fechas ISO, fusiona histórico + odds/stats + ELO; escribe:  
  - `out/laliga_enriched_all.csv`  
  - `out/laliga_enriched_model.csv`  
  Respuesta: resumen de filas por temporada, % missing, columnas.

- **POST `/api/v1/features/fixtures`**  
  Entrada: `fixtures.csv` y ventanas (`5,10`)  
  Acción: computa **features pre‑match** y escribe `out/fixtures_enriched.csv`.  
  Respuesta: columnas generadas + conteo de partidos.

- **POST `/api/v1/model/train`**  
  Entrada: `dataset_path` (por defecto `out/laliga_enriched_model.csv`), opciones (`use_xgb`, `calibration=platt|isotonic`)  
  Acción: entrenar, evaluar (TSS), calibrar y persistir `model.pkl`.  
  Respuesta: métricas globales + nombre del mejor modelo.

- **POST `/api/v1/predict`**  
  Entrada: `fixtures_enriched.csv` (o payload JSON)  
  Salida: `p_H, p_D, p_A` por partido + `elo_diff` + diferenciales de rolling y, si están, **odds** para comparación.

- **POST `/api/v1/odds/compare`**  
  Entrada: probabilidades del modelo + cuotas (`odds_avg` o `odds_close`)  
  Acción: calcula **probabilidades implícitas** (ajustadas por vigorish) y métricas comparadas (Brier/log_loss del mercado vs modelo).  
  Salida: tabla con **value bets** (si p_model × odd − 1 > umbral).

- **GET `/api/v1/health`** → `{"status":"ok"}`

> Genera automáticamente **OpenAPI.json** para que el frontend consuma la API tipada.
